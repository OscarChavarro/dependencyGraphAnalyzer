# Frontend

Angular 20 application for exploring dependency graphs generated by the backend. The UI renders Graphviz SVGs on a custom HTML5 canvas, keeps the graph snapshot in `sessionStorage`, and provides navigation, selection, relation-query, and group-editing tools.

## Capabilities

- Load configured cache, C/C++, and Java projects from the backend.
- Run `CACHE_LOADER`, `DEBIAN_PACKAGE_GENERATOR`, `CPP_SOURCES`, and `JAVA_SOURCES` generators.
- Render `structure.svg` and group SVGs with zoom, pan, and automatic centering.
- Navigate from structure nodes to `<group>.svg`.
- Cycle between groups with left/right arrow keys.
- Select nodes from the canvas or from the filterable side list.
- Open a contextual action menu on selected nodes.
- Query relations between two groups from the `structure` view.
- Highlight transitive dependencies or clients.
- Move nodes to another group through `POST /v1/moveNode` and refresh the graph.
- Show relations in a dialog with navigable links, checkboxes, and visual invalid-relation detection.
- Switch between Spanish and English with typed translation keys.

## Backend Integration

The backend base URL is injected through `BACKEND_BASE_URL`. In `src/main.ts`, the application attempts to read `/environment.json` with this shape:

```json
{
  "backend": {
    "url": "http://localhost:8080"
  }
}
```

If the file cannot be read, it uses `http://localhost:8080`. The fallback provider in `app.config.ts` is overridden by `main.ts` during the normal bootstrap path.

Consumed endpoints:

- `GET /v1/cachedProjects`
- `GET /v1/cppProjects`
- `GET /v1/javaProjects`
- `POST /v1/updateGraph`
- `POST /v1/enrichedEdges`
- `POST /v1/groupRelations`
- `POST /v1/moveNode`
- `GET /output/svg/{filename}`

## Structure

- `src/main.ts`: bootstrap, `environment.json` loading, and backend base URL injection.
- `src/app/app.ts`: main state coordinator, HTTP calls, and interaction flows.
- `src/app/app.html` and `src/app/app.sass`: side panel, generators, node list, and workspace layout.
- `src/app/model/`: graph types, API payloads, and menu models.
- `src/app/render/Html5CanvasGraphRenderer.ts`: SVG parser/renderer on canvas, node picking, zoom, pan, and color overrides.
- `src/app/gui/`: interaction techniques for keyboard, navigation, selection, and context menus.
- `src/app/localProcessing/`: local relation queries, flooding, and invalid-relation detection.
- `src/app/i18n/`: language state, translation service, and typed keys.
- `src/app/relation-dialog.component.ts`: relation dialog with links and per-line selection.
- `public/environment.json`: runtime configuration served as an asset.

## UI Model

The application keeps the graph snapshot in memory and in `sessionStorage` under `vsdk.graphModelSnapshot`. After successful graph generation:

1. It stores the snapshot returned by the backend.
2. It loads `structure.svg` from `/output/svg/structure.svg`.
3. It extracts interactive nodes from the SVG for the side list and node picking.
4. When entering a group, it loads `<group>.svg` and limits selection/listing to visible nodes.

For flooding, the UI uses `enrichedEdges` if they are already present in the snapshot. Otherwise it requests `/v1/enrichedEdges`; if that fails, it falls back to the local edge set.

## Commands

```bash
npm install
npm start
npm run build
npm test
```

`npm start` runs `ng serve --host 0.0.0.0`. The Angular `serve` target disables live reload.

## Local Development

1. Start the backend on `http://localhost:8080`.
2. Adjust `public/environment.json` if the backend runs on another host or port.
3. Start the frontend with `npm start`.
4. Use the generation panel selectors. Available data comes from `etc/projects/cachedProjects.json`, `etc/projects/cppProjects.json`, and `etc/projects/javaProjects.json` on the backend side.

The frontend does not generate graphs by itself: it always requires the backend to produce both the snapshot and the SVGs in `output/`.
